# 项目文档整理与下一步优化计划 — 设计说明

> 日期：2026-07-14  
> 状态：已批准（含审计修正）  
> 范围：文档信息架构重构 + 成文的分层路线图；**不**在本轮实现业务功能

## 背景与目标

北京中考升学指导网站（zhongzhao）阶段 0–4 已完成并落在 Cloudflare（D1 + Workers）。此后又交付了 AI 志愿助手、学校对比、FAQ、行政区网格筛选、倒计时增强、分数修复脚本等，但 `docs/HANDOFF.md` 仍停在 2026-07-13 的「阶段 4 ✅」，文档与代码脱节。

**目标（双读者）：**

1. 给 AI / 本人续会话：短交接入口，几分钟恢复上下文  
2. 给人：分类清楚的架构与运维说明，可独立查阅  
3. 产出带 **P0 / P1 / P2** 的全面优化路线图（产品 + 数据 + 工程运维）

## 方案选择

采用 **方案 B：模块化文档 + 独立路线图**（否决就地大杂烩修补与完整手册站）。

## §1 文档职责与真相源

每个主题只在一个文件写全，其它文件只链过去。

| 文件 | 写什么 | 不写什么 |
|------|--------|----------|
| `README.md` | 一句话定位、技术栈、最短快速开始、文档索引 | 详细运维、路线图细节、历史阶段打勾清单 |
| `docs/HANDOFF.md` | 短交接：线上 URL、资源 ID、近期变更摘要（5–10 条）、高频命令速查、新会话提示词 | 部署 step-by-step、功能百科、P0–P2 全文 |
| `docs/architecture.md`（新建） | 概念与流向：抓取→JSON→D1、运行时读路径、页面/功能地图、关键目录与脚本职责 | Cron 操作细节、域名绑定步骤、密钥操作 |
| `docs/cloudflare.md` | 操作手册：部署、本地/远程 D1 命令、Cron、监控告警、密钥 | 产品功能说明、路线图 |
| `docs/custom-domain.md` | 域名与 `SITE_URL` | 其它 |
| `docs/roadmap.md`（新建） | 基线完成说明 + 缺口盘点 + P0/P1/P2 + 非目标 | 操作手册正文 |

### 重叠消除（硬规则）

- **概念与数据流**：只写在 `architecture.md`  
- **可执行命令、Cron、告警、密钥**：只写在 `cloudflare.md`（域名相关在 `custom-domain.md`）  
- **HANDOFF**：仅表格式摘要 + 链接，不复制大段架构或部署正文  
- **环境变量 / secrets**（`env.example`、`CRON_SECRET`、`ALERT_WEBHOOK_URL` 等）：归属 `cloudflare.md`

### 真相源约定

| 主题 | 真相源 |
|------|--------|
| npm 脚本全集 | `package.json` `scripts` |
| HANDOFF 命令速查 | 仅 8–12 条人类高频命令；变更时对照 `package.json`，不追求与 scripts 一一枚举 |
| 线上 URL / Cloudflare 资源 ID | `docs/HANDOFF.md` |
| 架构与功能地图 | `docs/architecture.md` |
| 下一步做什么 | `docs/roadmap.md` |

## §2 现状盘点范围

盘点只服务：`architecture.md`（现在有什么）与 `roadmap.md`（缺什么 / 下一步）。不做代码大重构，不扩成产品 PRD。

### 纳入

1. **产品面**：现有路由与能力（首页倒计时、学校库、分数线、对比、政策、攻略/估分、FAQ、AI 助手、开放页等）  
2. **数据面**：抓取源、别名审计、`schools.json` 中间产物、本地/远程 D1、同步 Worker、修复脚本、健康指标含义  
3. **工程与运维面**：OpenNext 部署、两套 wrangler、health / Actions、告警、自定义域名与 `SITE_URL` 文档状态、`smoke:test` 等测试覆盖面  
4. **文档债务**：HANDOFF 滞后于 post–阶段 4 提交 → 「文档滞后清单」本身记入路线图（见 §3/§4 本次交付边界）

### 不纳入本次文档工作正文（可在 roadmap「非目标」提及）

重写视觉体系、换数据库、公开营销站、完整对外 API 平台。

### 状态标注规则（审计修正）

盘点条目的状态**只允许**下列标签，且需有证据：

| 标签 | 证据要求 |
|------|----------|
| `已上线` | 存在对应路由、Worker、或可引用的脚本/配置 |
| `文档未记载` | 代码/配置存在，但现行 docs 未描述 |
| `有风险信号` | 存在修复脚本，或近期 fix commit 明确针对该区域 |
| `未验证` | 缺少上述证据，或需访问线上才能确认（如自定义域名是否已绑定） |

**禁止**无证据标注「半完成」「已知坑」等主观状态。

### 旧阶段 0–4 叙事

压缩为「基线已完成（截至约 2026-07-13 阶段 4；其后功能见 architecture / 近期变更）」。明细删除或迁到 `roadmap.md` 附录，避免与 P0–P2 双轨。

## §3 P0 / P1 / P2 原则与方向

### 日历锚点（审计修正；今天为 2026-07-14，正值中招季窗口）

| 层级 | 时间窗 | 说明 |
|------|--------|------|
| **P0** | ≈ 至 2026-07 末（约 2 周） | 文档失真、数据可信、可观测性、季内在用关键缺陷 |
| **P1** | 2026 Q3（约至 9 月底） | 季后沉淀：体验与维护成本改进 |
| **P2** | 2027 中招季前 | 加深能力与自动化；可拆、可延后 |

**规则：** 凡影响 **2026 季内仍在使用** 的可信度或关键用户动线，优先进 P0/P1，不得塞进 P2「等明年」。

**同层排序：** 学生/家长受益面 × 实现成本 × 数据依赖风险；数据未稳的产品项不进 P0。

### 主题方向（写入 `roadmap.md` 时落成带成功标准的条目）

**P0**

- 按 §1 完成文档重构，对齐 post–阶段 4 能力  
- 数据质量：异常分数线规则、修复/回填流程**写清**；别名审计纳入例行说明  
- 运维基线：health / smoke / 告警 / 域名与 `SITE_URL` 在文档中可核对（未核验处标 `未验证`）

**P1**

- 产品：对比与筛选动线、估分/志愿助手与 AI 助手边界、移动端/微信内体验  
- 数据：抓取源稳定性与覆盖、同步失败可诊断性  
- 工程：smoke 扩展、脚本与查询层可维护性小改进

**P2**

- 更深填报辅助（仍非官方承诺）、政策/日程运营节奏  
- 性能与 SEO、域名/品牌固化、更完整监控  
- 「非目标」写清：换库、大重构、对外开放 API 平台等

### 条目写法

每条含：目标一句话、成功标准、触及面（产品/数据/工程）、依赖。禁止空泛「优化体验」。

### 与「本次交付」的关系（审计修正）

`roadmap.md` 中的 P0/P1/P2 是**后续实现队列**。  
**本轮（文档整理专项）只保证：文档体系落地 + 路线图成文。**  
文档债务清零（§1 结构与内容对齐）是本轮唯一必达的「P0 子集」；其余 P0 条目待 writing-plans / 后续实现，不在本轮改业务代码。

## §4 落地顺序与成功标准

### 本轮落地顺序

1. 对照路由、`package.json` scripts、近期提交、现有 docs 做盘点（遵守 §2 状态规则）  
2. 新建 `docs/architecture.md`、`docs/roadmap.md`  
3. 瘦身并更新 `docs/HANDOFF.md`  
4. 校准 `README.md`、`docs/cloudflare.md`（必要时小改 `custom-domain.md`）  
5. 自检：链接有效、命令与 scripts 不矛盾、路线图无 TBD 空壳、状态标签合规

### 本轮完成标准（可检查）

- [ ] `README` 有文档索引，能链到 HANDOFF / architecture / cloudflare / roadmap / custom-domain  
- [ ] `HANDOFF` 可指出至少 5 个线上相关 URL（或明确标注未使用）、至少 3 条启动/同步相关命令、并指向 architecture 与 roadmap  
- [ ] `architecture.md` 含数据流 + 当前功能地图（含 AI、对比、FAQ 等 post–阶段 4 能力）  
- [ ] `roadmap.md` 含日历锚定的 P0/P1/P2、条目成功标准、非目标；旧阶段 0–4 已压缩为基线说明  
- [ ] 无「无证据的半完成」类标注

### 本轮明确不做

- 不实现 roadmap 中除「文档债务」外的 P0/P1/P2 功能  
- 不大规模改业务代码（盘点若发现必须极小修正才能写准文档，另开条目，默认不做）  
- 不引入 docs 静态站 / 手册生成器（方案 C）

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 盘点臆造状态 | §2 四类标签 + 证据规则 |
| 文档再次漂移 | 真相源表 + HANDOFF 只链不抄 |
| P0 范围膨胀 | §3「本轮只出文档与路线图」硬边界 |
| 中招季时间误读 | §3 日历锚点写死到 2026-07 / Q3 / 2027 季前 |
