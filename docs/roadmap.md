## P1

季后沉淀（2026 Q3，约至 9 月底）：体验与维护成本改进，不影响季内必达项。

### P1-1 对比、筛选与助手边界

| 字段 | 内容 |
|------|------|
| **目标** | 理顺学校对比、行政区筛选、估分攻略与 AI 助手之间的入口与职责边界，减少用户混淆。 |
| **成功标准** | 各功能页有清晰互链与一句职责说明；对比页与筛选动线无需返回首页即可切换上下文；AI 助手不替代官方政策表述，有明确免责与跳转。 |
| **触及面** | 产品（`/compare`、`/schools`、`/guide`、`/assist`）。 |
| **依赖** | P0 文档对齐完成；季内流量高峰过后实施 UI 微调。 |

### P1-2 微信内与移动端体验

| 字段 | 内容 |
|------|------|
| **目标** | 优化微信内置浏览器与移动端的开放页引导、分享与核心查询动线。 |
| **成功标准** | `/open` 与 `in-app-browser` 逻辑在常见微信版本下可完成外链跳转或复制引导；首页与学校详情在窄屏下无关键信息遮挡。 |
| **触及面** | 产品（`/open`、全站响应式）。 |
| **依赖** | 真机微信环境回归；P0-3 域名与 `SITE_URL` 状态明确。 |

### P1-3 抓取诊断与 smoke 扩展

| 字段 | 内容 |
|------|------|

| **成功标准** | 同步失败可从 `/logs` 或文档化步骤定位到抓取源/别名缺口；`smoke:test` 覆盖对比、FAQ、assist API 等关键路径且纳入 CI 或例行手册。 |
| **触及面** | 数据（抓取、D1 写入）；工程（`workers/sync`、`scripts/smoke-test.mjs`）。 |

> 换库、视觉大改、营销站、对外开放 API 等见下文「非目标」，不纳入 P2 实施范围。

## 非目标

以下不在 P0–P2 队列内，本轮及近期均不推进：

- **换库**：D1 仍为唯一运行时数据库，不迁移至其它存储方案。
- **视觉大改**：不重做全站设计体系或大规模 UI 重构。
- **营销站**：不建设独立品牌/marketing 站点。
- **对外开放 API**：不提供面向第三方的公开 API 平台。
- **本轮业务代码**：除文档债务清零外，不实现 roadmap 中其它条目的业务功能改动；数据修复脚本的生产执行、告警/域名人工核验等另开周期。
